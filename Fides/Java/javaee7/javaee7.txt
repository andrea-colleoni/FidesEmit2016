1.Overview

Questo capitolo introduce allo sviluppo di applicazioni Java EE enterprise. Qui esaminerai le basi dello sviluppo, imparerai l'architettura Java EE e la sue API,
apprenderai importanti termini e concetti, e scoprirai come approcciarsi alla programmazione, assemblamento e impiego delle applicazioni Java EE.

Gli sviluppatori d'oggi sempre più riconoscono il bisogno di applicazioni distribuite, transazionali e portabili che fanno leva sulla velocità, sicurezza e affidabilità
di tecnologie lato server.

Le applicazioni enterprise forniscono la logica di buissness per un'impresa. Sono gestite centralmente e spesso interagiscono con altro software enterprise.

Nel mondo dell'information technology, le applicazioni enterprise devono essere progettate, sviluppate e prodotte con meno denaro, molta più velocità, e con poche risorse.

Con la piattaforma Java, Enterprise Edition (Java EE), lo sviluppo di applicazioni enterprise Java non è stato mai così semplice o veloce.

L'obiettivo della piattaforma Java EE è quello di fornire agli sviluppatori un insieme di potenti API che consentono di ridurre il tempo di sviluppo, ridurre la complessità dell'applicazione
e migliorarne le performance.

La piattaforma Java EE è sviluppata tramite la Java Community Process (JCP), responsabile di tutte le tecnologie Java.

Gruppi di esperti composti da parti interessate allo sviluppo hanno creato le Java Specification Requests (JSRs) per definire le varie tecnologie Java EE.

Il lavoro della Java Community sotto il programma JCP aiuta ad assicurare agli standard delle tecnologie Java stabilità e portabilità multi-piattaforma.

La piattaforma Java EE usa un modello semplificato di programmazione. L'uso di deployment descriptors XML (file di configurazione per un componente, modulo o applicazione enterprise)
sono opzionali.

Invece, uno sviluppatore può semplicemente inserire l'informazione come una 'annotazione' direttamente nel file sorgente Java, e il server Java EE configurerà i componenti a runtime.

Queste annotazioni sono generalmente usate per inserire in un programma dati che sarebbero altrimenti stati forniti con un deployment descriptor.

Con le annotazioni inserisci le informazioni nel tuo codice vicino all'elemento che deve essere annotato.


Nella piattaforma Java EE, la dependency injection può essere applicata a tutte le risorse di cui un componente ha bisogno, nascondendo di conseguenza la creazione e ricerca di queste risorse
direttamente dal codice dell'applicazione.

La dependency injection permette al container Java EE di inserire automaticamente riferimenti a altri componenti o risorse, usando le annotazioni.


Questo tutorial fa uso di esempi per descrivere le caratteristiche disponibili nella piattaforma Java EE per sviluppare applicazioni enterprise. 

Se sei un nuovo o un esperto sviluppatore enterprise, dovresti trovare gli esempi e il testo in accompagnamento una valida e accessibile conoscenza di base per creare le tue personali soluzioni.
